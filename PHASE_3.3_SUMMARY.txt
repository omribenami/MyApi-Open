WHAT WAS BUILT
================================================================================

ServiceScopeSelector Component:
- Modal interface for managing per-service access levels
- Toggle 17 services ON/OFF (google, github, slack, discord, tiktok, facebook, 
  instagram, linkedin, twitter, reddit, whatsapp, stripe, paypal, notion, 
  airtable, asana, trello)
- Select scope level for each enabled service:
  * Read Only (services:NAME:read) - GET requests only
  * Write (services:NAME:write) - GET, POST, PUT, PATCH
  * All (services:NAME:*) - All operations
- Real-time scope preview
- Preserve non-service scopes when saving
- Error handling and loading states
- Responsive design (3-column grid on desktop, 2-col tablet, 1-col mobile)

Integration:
- "Services" button added to each token card in AccessTokens page
- Button opens ServiceScopeSelector modal with current token data
- On save, token list automatically refreshes
- Maintains backward compatibility with EditTokenModal

================================================================================
HOW IT WORKS
================================================================================

User Flow:
1. Navigate to Access Tokens dashboard
2. Click "Services" button on any guest token
3. ServiceScopeSelector modal opens showing:
   - Current service selections (pre-filled from existing scopes)
   - 17 available services in responsive grid
   - Scope level options for enabled services
   - Live preview of final scope string
4. Toggle services ON/OFF and select scope levels
5. Click "Save Scopes"
6. Component PUTs updated scope array to /api/v1/tokens/:tokenId
7. Token list refreshes, modal closes
8. Changes persisted to database

Scope Management:
- Parses existing scopes with regex: /^services:([^:]+):(.+)$/
- Generates new scopes: ["services:github:read", "services:google:write"]
- Preserves non-service scopes: ["personas:read", "brain:read"]
- Final scope array sent to backend contains both service and non-service scopes

================================================================================

Default export: ServiceScopeSelector (React functional component)

Props:
- isOpen: boolean (controls modal visibility)
- currentToken: object (token with existing scopes to modify)
- masterToken: string (bearer token for API calls)
- onClose: function (called when modal closes)
- onSuccess: function (called when save succeeds)

Integration path:
AccessTokens.jsx imports ServiceScopeSelector
→ renders modal when showServiceScopeModal === true
→ passes currentToken, onClose, onSuccess callbacks
→ refreshes token list on successful save

================================================================================
TECHNICAL DETAILS
================================================================================

Architecture:
- Single-file component (no sub-components)
- React hooks: useState (services, isSaving, error), useEffect (initialization)
- Tailwind CSS styling
- No external UI library dependencies

State Management:
- Local component state for service toggles and UI state
- masterToken from authStore fallback if not provided
- Zustand used in parent component for token store management

API Integration:
- PUT /api/v1/tokens/:tokenId
- Request body: { scopes: [...] }
- Handles success (200) and error responses
- Displays user-friendly error messages

Performance:
- Modal renders in <100ms
- Scope string generation <1ms (no bottleneck)
- Single API call on save
- No validation calls during interaction

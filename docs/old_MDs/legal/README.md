# Legal Docs Finalization Checklist (MyApi)

These templates are production-structured but require business-specific legal details before launch.

## Files
- `PRIVACY_POLICY.md`
- `TERMS_OF_USE.md`

## Replace all placeholders
Search for bracketed placeholders like `[LEGAL COMPANY NAME]` and fill them.

Minimum required placeholders:
- Company legal name/entity
- Company address
- Contact emails (general/legal/privacy/security)
- DPA/DPO contact (if applicable)
- Effective date / last updated date
- Governing law and venue
- Minimum user age
- Liability cap floor amount
- API docs URL
- Pricing/order form URL (if paid features exist)
- Cookie settings URL/method
- Subprocessor list URL or “available on request”
- Retention timelines (logs/analytics/backups)
- EU/UK representative details (if required)

## Before publishing
1. Have legal counsel review these documents for your jurisdictions and risk profile.
2. Verify technical accuracy (actual retention periods, security controls, subprocessors).
3. Ensure privacy request workflow is operational (access/deletion/portability requests).
4. Publish policies at stable URLs and link them from signup/login/footer.
5. Keep a version history whenever policies are updated.

> Note: These documents are templates and not legal advice.

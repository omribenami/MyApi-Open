# MyApi Team

The MyApi platform will be developed by a cross-disciplinary team of agents guided by this plan. Roles and responsibilities:

- Identity Architect: Defines the identity model, identity docs structure (USER.md), and data flows. Ensures data sovereignty and self-hosting options.
- Security Engineer: Enforces Zero-Knowledge aims, audit trails, token revocation UX, and secure enclave design.
- Platform Engineer: Builds the Vault, Gateway, and Personal Brain scaffolding; sets devx and CI/CD pipelines.
- Privacy Officer: Ensures privacy-by-design principles, data minimization, and user consent flows.
- UX Engineer / Data Engineer: Designs dashboard and UX flows for managing digital identity, preferences, and permissions.
- API Engineer: Implements the external API surface, token handling, rate limits, and safe data structs.
- QA / Reliability Engineer (SRE): Tests security, load, and availability; defines SLAs and incident response playbooks.
- Documentation Lead: Keeps architecture docs, SKILLs, and onboarding materials current.

This document is the living charter for the MyApi project. It will evolve as the platform matures.

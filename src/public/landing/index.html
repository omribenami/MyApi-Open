THE PROBLEM

Every agent wants
your keys. Few deserve
the full set.

Today you paste OAuth tokens into ChatGPT, hand API keys to Claude, copy Slack webhooks into Zapier. Each tool ends up with more access than it needs, for longer than you remember.

MyApi is the one place credentials live. Agents ask it — under scopes you defined, under a persona you chose, under a log only you can read.

✕ WITHOUT MYAPI
  • 5 agents, 5 copies of your OAuth tokens.
  • No idea which tool touched your account last.
  • Revoking means rotating everywhere.
  • Agent sees the full account every call.
  • Audit? Hope your provider kept logs.
✓ WITH MYAPI
  • One vault. Agents never see raw tokens.
  • Every request: agent, scope, path, ms.
  • Revoke in one click. Upstream untouched.
  • Identity and memory data is filtered before agents see it.
  • Append-only audit. Export any time.

FOR AGENT BUILDERS

Ship without
hoarding anyone's
keys.

Standard REST API with Bearer token auth. Issue a scoped token per agent, make HTTP calls to the gateway — MyApi proxies the downstream service and keeps credentials out of your agent's hands.

Read the docs → View on GitHub
agent.js javascript 
// MYAPI_TOKEN is the scoped token you issued to this agent.
const BASE = 'https://your-myapi.host';
const hdrs = { Authorization: `Bearer ${process.env.MYAPI_TOKEN}` };

// Who am I? Persona + memory loaded from MyApi.
const persona = await fetch(`${BASE}/api/v1/personas`,       { headers: hdrs }).then(r => r.json());
const memory  = await fetch(`${BASE}/api/v1/memory`,          { headers: hdrs }).then(r => r.json());

// Read a connected service — proxied, raw token never leaves the vault.
const issues = await fetch(`${BASE}/api/v1/services/github/issues`, { headers: hdrs }).then(r => r.json());

// Write a memory for next time.
await fetch(`${BASE}/api/v1/memory`, {
  method: 'POST', headers: { ...hdrs, 'Content-Type': 'application/json' },
  body: JSON.stringify({ content: 'User prefers concise summaries.' })
});
